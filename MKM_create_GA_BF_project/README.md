# MKM_create_GA_BF_project

Это проект в котором используется генетический алгоритм (GA) и метод грубого перебора (BF) для построение минерально-компонентной модели по комплексу ГИС в составе которого ядерные методы, СГК, ННК, ГК:

1. посчитать МКМ по всем LAS-файлам (BF + GA);
2. построить финальные литологические колонки для отчёта.

Результаты, ради которых всё запускается, лежат в `outputs/`.

## Что здесь есть

- `run_GA_BF_all_wells.py` — основной пакетный запуск расчётов по всем скважинам с помощью грубого перебора и с помощью генетического алгоритма.
- `build_final_lithology_columns.py` — переводит результаты GA в итоговые литологические классы и графические колонки.
- `mkm_core.py`, `mkm_interval_optimizer.py`, `mkm_ga_engine.py`, `mkm_bruteforce_engine.py`, `scale.py` — вычислительное ядро.
- `config/` — файлы ограничений и сеток для подбираемых петрофизических матриц (`a_min_*.in`, `a_max_*.in`, `a_k_*.in`).
- `data/las/` — входные LAS.
- `outputs/mkm_result/` — результаты расчёта МКМ по скажинам.
- `outputs/final_lithology/` — финальные литологические колонки.
- `requirements.txt` — зависимости Python.

## Как запустить локально

Откройте терминал в папке `MKM_create_GA_BF_project`.

### 1) Подготовить окружение

Установка зависимостей:

```bash
pip install --upgrade pip
pip install -r requirements.txt
```

### 2) Посчитать МКМ по всем скважинам и с помощью генетического алгоритма и с помощью перебора

```bash
python run_GA_BF_all_wells.py
```

По умолчанию:

- берутся LAS из `data/las`;
- выход пишется в `outputs/mkm_result`;
- используются конфиги из `config`;
- выполняются оба этапа: BF и GA.

Полезные варианты:

```bash
# Только GA, BF не пересчитывать
python run_GA_BF_all_wells.py --ga-only

# Только выбранные скважины
python run_GA_BF_all_wells.py --only 621_1700_1780 6034_1703_1796
```

### 3) Построить финальную литологию

```bash
python build_final_lithology_columns.py
```

По умолчанию:

- вход: `outputs/mkm_result`;
- выход: `outputs/final_lithology`.

Пример только для части скважин:

```bash
python build_final_lithology_columns.py --only 621_1700_1780 6034_1703_1796
```

## Что должно появиться после запуска

Для каждой скважины в `outputs/mkm_result/<well>/`:

- `*_mkm_bf.png`, `*_mkm_ga.png`;
- `*_intervals_bf.npz`, `*_intervals_ga.npz`;
- `*_mkm_ga_plot_data.npz`;
- `metrics.txt`.

Для каждой скважины в `outputs/final_lithology/<well>/`:

- `*_final_lithology_points.csv`;
- `*_final_lithology_intervals.csv`;
- `*_final_lithology_column.png`.

Общие итоговые файлы:

- `outputs/final_lithology/final_lithology_summary.csv`;
- `outputs/final_lithology/final_lithology_overview.png`.

## Все CLI-параметры скриптов

Ниже перечислены параметры двух основных скриптов из этой папки.

Быстрый способ посмотреть их прямо в терминале:

```bash
python run_GA_BF_all_wells.py --help
python build_final_lithology_columns.py --help
```

### Параметры `run_GA_BF_all_wells.py`

- `--ga-only` (флаг, по умолчанию выключен)  
  Запускает только GA и не пересчитывает BF. Удобно, когда BF уже посчитан и нужно быстро обновить GA.

- `--project-root` (по умолчанию: авто, папка скрипта)  
  Корень проекта, относительно которого разрешаются остальные пути.

- `--las-dir` (по умолчанию: `data/las`)  
  Папка, где лежат входные `.las` файлы.

- `--out-root` (по умолчанию: `outputs/mkm_result`)  
  Папка для результатов расчёта по каждой скважине.

- `--config-dir` (по умолчанию: `config`)  
  Папка с матрицами ограничений и сетки (`a_min_*`, `a_max_*`, `a_k_*`).

- `--depth` (по умолчанию: `DEPT`)  
  Мнемоника кривой глубины в LAS.

- `--litho` (по умолчанию: `LITO`)  
  Мнемоника кривой литотипа в LAS.

- `--props MNEM1 MNEM2 MNEM3 MNEM4` (по умолчанию: авто)  
  Четыре кривые-свойства для МКМ. Если не указывать, используются `POTA THOR RHOB WNKT`.

- `--a-min-coll` (по умолчанию: `a_min_coll.in`)  
  Имя файла с нижними границами матрицы коллектора.

- `--a-max-coll` (по умолчанию: `a_max_coll.in`)  
  Имя файла с верхними границами матрицы коллектора.

- `--a-k-coll` (по умолчанию: `a_k_coll.in`)  
  Имя файла сетки (количество узлов) для BF по матрице коллектора.

- `--a-min-glin` (по умолчанию: `a_min_glin.in`)  
  Имя файла с нижними границами матрицы глин.

- `--a-max-glin` (по умолчанию: `a_max_glin.in`)  
  Имя файла с верхними границами матрицы глин.

- `--a-k-glin` (по умолчанию: `a_k_glin.in`)  
  Имя файла сетки (количество узлов) для BF по матрице глин.

- `--w-negative` (по умолчанию: `0.8`)  
  Вес штрафа за отрицательные доли компонент в целевой функции.

- `--w-glin` (по умолчанию: `0.1`)  
  Вес «плохой доли» для глинистых интервалов.

- `--w-coll` (по умолчанию: `0.1`)  
  Вес «плохой доли» для коллекторных интервалов.

- `--splitting` (по умолчанию: `4`, допустимо: `2/3/4/5`)  
  Плотность сетки BF для тех ячеек `a_k`, где значение не равно 1.  
  Чем больше значение, тем точнее, но тем дольше расчёт.

- `--population-size` (по умолчанию: `400`)  
  Размер популяции GA в каждом поколении.

- `--ngen` (по умолчанию: `10000`)  
  Верхний лимит по числу поколений GA. Обычно остановка происходит раньше по `--patience`.

- `--cxpb` (по умолчанию: `0.8`)  
  Вероятность кроссовера (скрещивания) в GA.

- `--mutpb` (по умолчанию: `0.1`)  
  Вероятность мутации особи в GA.

- `--indpb` (по умолчанию: `0.5`)  
  Вероятность мутации каждого отдельного гена внутри особи.

- `--tournsize` (по умолчанию: `2`)  
  Размер турнира при селекции в GA.

- `--patience` (по умолчанию: `50`)  
  Число поколений без улучшений, после которого включается ранняя остановка.

- `--min-delta` (по умолчанию: `1e-7`)  
  Минимальное улучшение метрики, которое считается «реальным прогрессом».

- `--n-jobs` (по умолчанию: `1`)  
  Количество процессов для параллельной оценки fitness в GA.

- `--seed` (по умолчанию: `4`)  
  Инициализация генератора случайных чисел для воспроизводимости.

- `--only WELL1 WELL2 ...` (по умолчанию: не задано)  
  Ограничивает расчёт только указанными скважинами (по имени файла без `.las`).

- `--continue-on-error` (флаг, по умолчанию выключен)  
  Если одна скважина упала с ошибкой, скрипт продолжит обработку остальных.

- `--verbose` (флаг, по умолчанию выключен)  
  Подробный лог BF/GA по интервалам и итерациям.

### Параметры `build_final_lithology_columns.py`

- `--project-root` (по умолчанию: авто, папка скрипта)  
  Корень проекта для разрешения относительных путей.

- `--input-root` (по умолчанию: `outputs/mkm_result`)  
  Папка, где лежат подпапки скважин с файлами `*_mkm_ga_plot_data.npz`.

- `--output-root` (по умолчанию: `outputs/final_lithology`)  
  Папка, куда будут сохранены финальные CSV и PNG.

- `--only WELL1 WELL2 ...` (по умолчанию: не задано)  
  Строит финальную литологию только для перечисленных скважин.
