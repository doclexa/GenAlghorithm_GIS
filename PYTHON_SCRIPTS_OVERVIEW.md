# Описание Python-скриптов проекта

Ниже перечислены все `.py`-файлы, найденные в проекте, с кратким назначением каждого.

## Ядро расчета МКМ и оптимизации

- `mkm_core.py` — базовый модуль проекта: загрузка LAS, выбор кривых, расчет МКМ, метрики качества, расчет `Q`, разбиение на литотип-интервалы, построение графиков и утилиты валидации/путей.
- `mkm_ga_engine.py` — реализация генетического поиска (DEAP) для матриц `A_coll` и `A_glin`: функции fitness, bounded crossover/mutation, early stopping (`patience`), сбор статистики по поколениям.
- `mkm_bruteforce_engine.py` — движок полного перебора: построение сеток значений матриц и поиск лучшей матрицы для коллекторов/глин с учетом вырожденных матриц.
- `mkm_interval_optimizer.py` — интервальная оптимизация по непрерывным литотип-интервалам: orchestration BF/GA по интервалам, сбор итоговой МКМ, итоговые метрики, quality-curve по поколениям, экспорт CSV/NPZ.
- `ga_fitness.py` — вспомогательный модуль fitness для GLIN (с worker init), используется как легковесная функция оценки в стиле multiprocessing.

## CLI-скрипты основного пайплайна

- `mkm_run_bruteforce.py` — основной CLI для интервального brute force: читает LAS и конфиги, запускает интервальный BF, сохраняет матрицы, CSV по интервалам, график МКМ и (опционально) `.npy`.
- `mkm_run_ga.py` — основной CLI для интервального генетического поиска: аналог BF-CLI, но с `GAParams` и сохранением результатов интервального GA.
- `mkm_interval_optimizer.py` — помимо библиотечных функций, фактически является core-слоем, который вызывают оба CLI (`mkm_run_bruteforce.py`, `mkm_run_ga.py`).
- `calc_mkm_lab.py` — CLI расчета МКМ по фиксированным лабораторным матрицам (без оптимизации): считает метрики и строит итоговый график по глубине.
- `scripts/mkm_bruteforce.py` — тонкая точка входа-обертка для запуска `mkm_run_bruteforce.main()` из папки `scripts`.
- `scripts/mkm_ga.py` — тонкая точка входа-обертка для запуска `mkm_run_ga.main()` из папки `scripts`.

## Тюнинг и исследование гиперпараметров GA

- `tune_mkm_gen_hyperparams.py` — случайный поиск гиперпараметров GA с критериями по времени и качеству относительно baseline; сохраняет отчет trial-ов, лучшие матрицы и график лучшей модели.
- `analyze_ga_hyperparams.py` — sweep по гиперпараметрам GA (population/cxpb/mutpb/tournsize/indpb) для классического (не интервального) сценария; сохраняет кривые, summary CSV и графики влияния.
- `study_interval_ga_hyperparams.py` — строгий sweep гиперпараметров именно для интервального GA; строит кривые `Q` по поколениям, summary CSV, поддерживает режим `--plot-only`.

## Скрипты визуализации и утилиты графиков

- `scripts/plot_mkm_from_matrices.py` — строит график МКМ по глубине из LAS и двух заранее заданных матриц (`matrix_coll`, `matrix_glin`).
- `scripts/plot_bad_bf_by_q.py` — подбирает BF-вариант, наиболее близкий к заданному `target Q`, и сохраняет соответствующий график МКМ.
- `experiments/plot_tune_tradeoff.py` — scatter-график «время поиска vs Q» по CSV отчета тюнинга (`tune_mkm_gen_hyperparams.py`).

## Эксперименты сравнения BF и GA (папка `experiments`)

- `experiments/__init__.py` — маркер пакета и краткое описание набора экспериментальных скриптов.
- `experiments/compare_bf_ga_skv621.py` — главный эксперимент сравнения интервальных BF и GA: запускает серии (full BF, budget-matched BF, coarse BF, GA), сохраняет benchmark и интервальные CSV сравнения.
- `experiments/plot_experiment_results.py` — строит набор сравнительных графиков по CSV из `compare_bf_ga_skv621.py` (качество, время/качество, evals/качество, локальные дельты по интервалам).
- `experiments/build_diploma_report.py` — собирает итоговый Markdown-отчет для диплома на основе CSV бенчмарка и интервальных сравнений.
- `experiments/plot_quality_vs_tested_matrices.py` — отдельный эксперимент/график зависимости `Q` от количества опробованных матриц (серии `bf_N` против `ga`) с экспортом CSV+PNG.
- `experiments/plot_stability_q.py` — анализ устойчивости `Q` при сдвиге границ поиска: обновляет stability CSV (в т.ч. синтез `bf_5` из `bf_4`) и строит график стабильности.

## Примечание по структуре

- В проекте есть «двойной» слой запуска: библиотечные модули (`mkm_*`) и легкие entrypoint-скрипты в `scripts/`.
- Основная вычислительная логика сосредоточена в `mkm_core.py`, `mkm_ga_engine.py`, `mkm_bruteforce_engine.py`, `mkm_interval_optimizer.py`.
- Папка `experiments` содержит воспроизводимые исследования и построение отчетных артефактов для сравнения методов.
